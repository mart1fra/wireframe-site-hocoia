# Arborescence du site Hocoia

## Objectifs du site
- Transformer un site vitrine en plateforme de services
- Optimiser acquisition (SEO national + local + international)
- Améliorer conversion (B2B + B2C + RH santé)
- Clarifier les parcours utilisateurs
- Asseoir la crédibilité (expertise médicale + tech)

## Principes structurants
Architecture en 3 couches :
1. **Acquisition** (SEO & GEO) — pages expertises, pages locales
2. **Conversion** (pages business) — solutions par cible
3. **Contenu & réassurance** (hub + FAQ) — ressources, à propos, presse

## Logique UX clé
- Orientation immédiate dès la home via un sélecteur de profil
- 3 profils : Organisations / Professionnels de santé / Patients
- Parcours distincts mais connectés entre eux

## Mapping des parcours
- **Décideurs (B2B)** : problématique → Solution Hocoia → Étude de cas / Vidéothèque → Téléchargement LB / RDV
- **Soignant (RH)** : Vision métier → Offres → Dépôt de document / Disponibilités → Devenir soignant Hocoia
- **Patient (B2C)** : Recherche locale → Prise de RDV / Demande de passage dans ma ville

## Pages spéciales

### Pages GEO (SEO local)
Création automatique de pages locales pour chaque ville/étape.
Exemple : "dépistage cancer du sein Bourges"
- Générateur de pages dynamiques avec import BDD
- Pages optimisées SEO local + maillage interne
- Quand on clique sur une ville → redirige vers l'HocoApp

### Dimension internationale
- Architecture multi-instance multilingue
- Répertoires : hocoia.com/fr/page et hocoia.com/en/page
- Hreflang pour le SEO international
- Connecteur traduction automatique avec validation avant publication
